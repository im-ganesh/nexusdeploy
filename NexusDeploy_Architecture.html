CI/CD Pipeline Stages
Stage 1

Source code commit

Developer pushes code to GitHub repository, triggering the automation chain.

Stage 2

Webhook trigger

GitHub webhook fires automatically, launching the Jenkins pipeline — no manual steps.

Stage 3

Build application

Jenkins compiles and packages the application artifact.

Stage 4

Code quality (SonarQube)

Scans for bugs, code smells, vulnerabilities, and maintainability issues.

Stage 5

Security scan (Trivy)

Scans container and dependencies for known CVEs and security risks.

Stage 6

Docker image build

Application is packaged into an immutable Docker container image.

Stage 7

Push to Amazon ECR

Image is stored securely in the encrypted private container registry.

Stage 8

Deploy to ECS Fargate

Latest container version is deployed automatically as a serverless task.

Stage 9

Monitoring

CloudWatch tracks logs, metrics, health status, and fires alerts.
